Sequence Alignment Tool

Raji Sundararajan, Manoj Hoskeri, Joel Menezes

Abstract

In Bioinformatics, a sequence alignment is a way of arranging the sequences of DNA, RNA, or protein to identify regions of similarity that may be a consequence of functional, structural, or evolutionary relationships between the sequences. Our project aims to design and build a user-friendly interactive web application which will facilitate in demonstrating the concepts of sequence alignment.

Global sequence alignment will be done using the Needleman-Wunsch algorithm and local sequence alignment will be done using Smith-Waterman algorithm. The application will teach the user about the various steps involved in the process of sequence alignment. It will also allow the user to provide answers and inputs. In this manner, the user gets a more hands-on learning experience and makes the overall teaching process more interesting.

Plan of Action

The plan is to build a web application which enables the users to understand sequence alignment techniques in the BioInformatics domain. A brief explanation of the premise of the Needleman-Wunsch algorithm and Smith-Waterman algorithm would be provided with links to more detailed explanations.

We will make use of simple test cases to illustrate the step by step working of the algorithms. Once the user is familiar with how to achieve sequence alignment, the user can try to solve sequence alignment problems on an interactive page. The user can fill up values of the matrix used in the Dynamic programming approach.

Given two sequences of length about 15-20 characters (this is to keep it simple), the application would bring up a matrix indicating rows and columns. The user needs to fill up the matrix entries. Based on the user entries in the matrix, the application would indicate whether the value entered in the cell is correct or not by highlighting the cell with colors green or red.

There will be different levels of solving the problems. The initial levels may have most of the cell values filled up and challenge the user to fill up only a few values. The application would provide certain hints at calculating the next matrix value. As the levels go higher up, the user must enter all the matrix values without any help from the application.

Once the user has completed filling up all the matrix entries, a correctness check would be made and a score would be assigned. The application would display all possible sequence alignments once all the matrix entries are correct. The application will award users with stars based on their achievements and their previous games would be persisted. We could also maintain a leaderboard of the registered users to show the highest scores.
